# Project Conversation Rules

## Explanation Style

- When answering technical questions, explain the problem itself first.
- Then, whenever it helps understanding, use frontend knowledge to provide analogies and intuition.
- Prefer frontend analogies from React state, component tree, routing, request/response flow, event flow, async rendering, and client/server boundaries.
- Keep the analogy as a bridge for understanding, but do not replace the accurate technical explanation.
- When discussing LangChain, LangGraph, middleware, memory, streaming, tools, or agent workflows, proactively relate them to familiar frontend architecture concepts when appropriate.
