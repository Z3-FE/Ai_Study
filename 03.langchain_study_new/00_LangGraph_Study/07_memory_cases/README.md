# LangGraph 记忆学习路线（由简到难）

1. InMemorySaver 单线程短期记忆  
同一 `thread_id` 记住上下文，换线程丢失。

2. InMemorySaver + stream_mode  
把短期记忆和流式输出结合，理解 `values` / `updates` / `messages` 差异。

3. InMemorySaver + InMemoryStore  
同进程内做“跨线程共享记忆”（不落盘）。

4. Postgres 持久化 + 多用户隔离  
`user_id` / `thread_id` 分层，支持真实多会话。

5. Postgres + 向量检索（轻量 Memory-RAG）  
用户长期记忆入库，结构化资料走 KV，自然语言记忆走语义检索，召回最相关记忆后再注入上下文。
补充实现案例：`06_记忆四层流水线_demo.py`，展示“原始输入采集 -> 提炼 -> 存储分流 -> 治理”的完整处理思路。

6. 记忆治理（压缩/过期/权限）  
总结摘要、TTL 清理、敏感字段隔离、可审计。
也就是：记忆不是存进去就结束了，还要考虑怎么提炼、去重、更新、过期、脱敏，以及哪些信息允许长期保留。

7. 生产级混合架构  
短期（`checkpointer`）+ 长期（DB）+ 语义记忆（vector）+ 中断恢复 + 观测（LangSmith）。
也就是：真实项目通常不会只靠一种记忆，而是把会话状态、用户资料、长期记忆、知识检索、恢复能力和观测能力组合成一整套线上架构。
补充实现案例：`07_生产级混合架构_demo.py`，展示“会话层 + 用户层 + 业务层 + 治理层”如何在一个聊天型 graph 中协作。
