# 人机交互（HCI）学习清单

这份清单的目标不是只学习传统的人机交互理论，而是结合你现在正在学习的：

- 聊天产品
- Agent 系统
- LangGraph / LangChain
- Memory / Tool / Streaming / Interrupt
- 前端交互设计

也就是：**从 AI Agent 产品的实际开发视角，学习人机交互。**

---

## 1. 学习目标

学完这一部分，希望你能逐步回答这些问题：

- 为什么有些 AI 产品“能用”，但用户还是觉得不好用？
- 用户为什么会误解 Agent 当前在做什么？
- 什么时候应该自动执行，什么时候应该让用户确认？
- 工具调用、记忆、检索、流式输出，应该怎么展示才不让人困惑？
- 为什么有些 AI 产品会让人“没有掌控感”？
- 一个好的 Agent 产品，交互上到底好在哪里？

---

## 2. 推荐学习顺序

### 2.1 HCI 基础

先建立“人机交互到底在解决什么问题”的底层认知。

建议学习：

1. 可用性（Usability）
2. 反馈（Feedback）
3. 一致性（Consistency）
4. 可见性（Visibility）
5. 心智模型（Mental Model）
6. 错误预防与错误恢复（Error Prevention / Recovery）
7. 用户控制权（User Control）

你要重点理解：

- 系统做了什么，用户能不能看懂？
- 用户下一步该做什么，系统有没有明确提示？
- 用户犯错时，系统有没有办法帮他纠正？

---

### 2.2 对话式交互

这一部分和聊天产品、AI 助手非常相关。

建议学习：

1. 聊天式界面的优点与局限
2. 单轮对话与多轮对话的区别
3. 对话上下文的组织方式
4. 系统回复的可理解性
5. 流式输出带来的反馈感
6. 对话中的中断、澄清、确认

你要重点理解：

- 为什么“会聊天”不等于“好交互”
- 聊天 UI 容易掩盖系统真实状态
- 多轮上下文越强，越容易让用户误以为系统“什么都记得”

---

### 2.3 AI / Agent 交互设计

这部分是和你现在学习最相关的核心区域。

建议学习：

1. Agent 当前状态如何反馈给用户
2. Tool call 如何展示
3. Memory 如何解释给用户
4. Retrieval / 搜索结果如何展示
5. Interrupt / Approval 何时触发
6. 多步骤任务如何可视化
7. 不确定性如何表达

你要重点理解：

- Agent 的“能力”不等于“用户的信任”
- 用户是否知道：
  - 当前用了什么工具
  - 当前记住了什么
  - 当前为什么这样回答
  - 当前哪里是不确定的

---

### 2.4 人机协作（Human-in-the-loop）

这是 Agent 产品非常重要的一部分。

建议学习：

1. 什么场景适合全自动
2. 什么场景必须让用户确认
3. 什么场景允许用户中断
4. 什么场景应该支持撤销 / 回滚
5. 怎么让用户“接管”系统

你要重点理解：

- 人机协作不是“用户一直点确认”
- 真正好的协作是：
  - 系统自动做擅长的事
  - 用户只在关键点介入

---

### 2.5 真实 AI 产品交互分析

建议挑几个产品做拆解：

1. ChatGPT
2. Claude
3. Cursor
4. GitHub Copilot
5. Perplexity
6. assistant-ui / 聊天类前端框架

拆解时建议观察：

1. 输入区怎么设计
2. 历史线程怎么组织
3. Tool / 搜索 / 推理状态怎么展示
4. 流式输出怎么给反馈
5. 错误提示怎么写
6. 用户如何中断、重试、继续

---

## 3. 结合 Agent 学习时最值得关注的主题

如果从你当前项目出发，我建议重点盯这几个话题：

### 3.1 Memory 的交互问题

用户最容易困惑：

- 你到底记住了什么？
- 你是当前会话记住，还是长期记住？
- 你为什么突然叫对我的名字了？
- 你是不是把不该记的也记了？

要学习的重点：

- 如何让“记忆”被用户感知
- 如何解释短期记忆与长期记忆
- 如何提供删除/修改/确认机制

---

### 3.2 Tool 的交互问题

用户最容易困惑：

- 你现在是在思考，还是在调用工具？
- 工具调用失败了为什么不告诉我？
- 你到底查了什么数据？

要学习的重点：

- Tool 调用状态展示
- Tool 结果展示
- Tool 失败反馈
- Tool 权限与确认

---

### 3.3 Streaming 的交互问题

用户最容易感受到的是“系统有没有在工作”。

要学习的重点：

- 流式输出为什么能提升响应感
- 什么时候只流最终答案
- 什么时候还要流中间步骤
- 怎么避免把内部 JSON / 中间思考全暴露给用户

---

### 3.4 Interrupt / Approval 的交互问题

用户是否愿意信任 Agent，很大程度上取决于：

- 高风险操作前是否确认
- 用户能不能打断系统
- 用户能不能修改 agent 计划

要学习的重点：

- 什么操作必须确认
- 什么操作可以自动执行
- 什么阶段应该允许“继续 / 停止 / 重试”

---

## 4. 推荐学习路径（按周 / 按阶段）

### 第一阶段：建立概念

目标：

- 先理解 HCI 基础概念
- 能用自己的话解释：
  - 可用性
  - 反馈
  - 心智模型
  - 用户控制权

建议输出：

- 写一份自己的术语笔记
- 用聊天产品举例解释每个概念

---

### 第二阶段：结合聊天产品观察

目标：

- 不只学概念，而是拿真实产品观察

建议练习：

1. 打开 ChatGPT / Claude / Cursor
2. 观察：
   - 回复时系统给了什么反馈
   - 哪些状态是清楚的，哪些状态是模糊的
   - 哪些设计让你更安心，哪些让你不安心

建议输出：

- 一份产品体验分析表

---

### 第三阶段：结合你自己的 Agent Demo

目标：

- 把 HCI 视角真正用到你现在的案例里

建议观察对象：

1. `06_1_记忆四层流水线_demo.py`
2. `06_2_手写版.py`
3. `07_生产级混合架构_demo.py`

建议思考：

- 这个 demo 里哪些内容应该暴露给用户？
- 哪些是内部消息，不该显示？
- memory 写入时，用户该不该知道？
- tool / retrieval / reasoning 该怎么反馈？

---

### 第四阶段：做交互重构

目标：

- 从“能跑”升级到“更像真实产品”

建议做的改造：

1. 给 memory 写入加用户可见提示
2. 给 tool 调用加状态反馈
3. 给最终回答和中间步骤做分层展示
4. 设计 interrupt / approval 交互
5. 给长期记忆增加“查看 / 删除 / 修改”入口

---

## 5. 建议重点掌握的术语

建议优先掌握这些词：

1. HCI（Human-Computer Interaction）
2. Usability（可用性）
3. Feedback（反馈）
4. Mental Model（心智模型）
5. Affordance（可供性）
6. User Control（用户控制权）
7. Error Recovery（错误恢复）
8. Transparency（透明性）
9. Explainability（可解释性）
10. Human-in-the-loop（人类参与闭环）
11. Interrupt（中断）
12. Approval（确认）
13. Trust（信任）
14. Cognitive Load（认知负担）
15. Context Awareness（上下文感知）

---

## 6. 这部分学习和你当前 Agent 学习的关系

你现在学的这些技术：

- LangGraph
- memory
- checkpoint
- store
- tool
- streaming
- interrupt

如果没有 HCI 视角，很容易变成：

- 只知道“怎么实现”
- 但不知道“为什么用户会觉得不好用”

所以这部分学习的价值在于：

**把“会写 agent”升级为“能设计一个更像真实产品、用户更能理解的 agent”。**
