# Olist 电商数据指标口径文件资料

## 1. 文档定位

本文档用于第一阶段 Olist 电商数据源的语义层建设，作为后续 `metric_definitions`、`join_rules`、`business_rules`、`sql_templates` 的基础资料。

适用场景：

- 自然语言问数时检索指标口径。
- Text-to-SQL 生成前约束表、字段、过滤条件和 Join 方式。
- SQL 审核时校验指标是否按统一口径计算。
- 前端指标口径页展示和人工维护。
- 文档导入语义层时作为人工确认基准。

数据源：

| 项目 | 内容 |
| --- | --- |
| 数据集 | Brazilian E-Commerce Public Dataset by Olist |
| PostgreSQL 数据库 | `enterprise_data_ai` |
| PostgreSQL schema | `olist` |
| 原始数据目录 | `data/raw/olist/` |
| 业务时间范围 | 约 2016-2018 年订单数据 |
| 当前定位 | 第一阶段交易分析主数据源 |

## 2. 核心数据表

| 表名 | 类型 | 主要用途 | 主键 / 粒度 |
| --- | --- | --- | --- |
| `olist.orders` | 事实表 | 订单状态、下单、审批、发货、送达、预计送达时间 | `order_id`，一行一个订单 |
| `olist.order_items` | 事实表 | 订单商品明细、商品价格、运费、卖家 | `order_id + order_item_id`，一行一个订单商品行 |
| `olist.order_payments` | 事实表 | 支付方式、分期数、支付金额 | `order_id + payment_sequential`，一行一个支付记录 |
| `olist.order_reviews` | 事实表 | 评价分、评价内容、评价创建和回复时间 | `review_id`，一个订单可能有多条评价记录 |
| `olist.customers` | 维表 | 客户匿名 ID、城市、州、邮编 | `customer_id`，一行一个订单客户记录 |
| `olist.products` | 维表 | 商品品类、名称长度、描述长度、图片数、重量、尺寸 | `product_id`，一行一个商品 |
| `olist.sellers` | 维表 | 卖家城市、州、邮编 | `seller_id`，一行一个卖家 |
| `olist.geolocation` | 维表 | 邮编前缀、经纬度、城市、州 | 邮编前缀粒度，可能一对多 |
| `olist.product_category_translation` | 维表 | 商品品类葡萄牙语到英语映射 | `product_category_name` |

## 3. 通用业务规则

### 3.1 默认订单范围

多数交易分析默认只统计已完成履约订单：

```sql
orders.order_status = 'delivered'
```

原因：

- `delivered` 表示订单已实际送达，适合分析真实成交、物流、评价。
- 取消、不可用、处理中订单会干扰销售额、配送时长和评价分析。

例外：

| 分析类型 | 订单状态口径 |
| --- | --- |
| 订单状态分布 | 不过滤订单状态 |
| 取消率 / 异常订单分析 | 需要包含 `canceled`、`unavailable` 等状态 |
| 下单趋势 / 用户购买意向 | 可统计全部订单，也可对比 `delivered` 订单 |
| 支付方式分析 | 默认按有支付记录订单统计，可结合订单状态说明 |

### 3.2 默认时间字段

| 业务问题 | 默认时间字段 | 说明 |
| --- | --- | --- |
| 销售趋势、订单趋势、品类排行 | `orders.order_purchase_timestamp` | 用户下单时间 |
| 支付趋势 | `orders.order_purchase_timestamp` | Olist 支付表无独立支付时间，默认使用下单时间 |
| 发货耗时 | `orders.order_approved_at` 到 `orders.order_delivered_carrier_date` | 从审批到交给物流 |
| 配送耗时 | `orders.order_delivered_carrier_date` 到 `orders.order_delivered_customer_date` | 从交给物流到客户签收 |
| 履约总时长 | `orders.order_purchase_timestamp` 到 `orders.order_delivered_customer_date` | 从下单到签收 |
| 评价趋势 | `order_reviews.review_creation_date` | 评价创建时间 |

### 3.3 金额字段说明

| 字段 | 含义 | 是否包含运费 |
| --- | --- | --- |
| `order_items.price` | 商品行销售价格 | 不包含 |
| `order_items.freight_value` | 商品行运费 | 是运费本身 |
| `order_payments.payment_value` | 订单支付金额 | 通常包含商品、运费及可能的支付拆分金额 |

注意：

```text
销售额、GMV、支付金额不能混用。
当用户只说“销售额”时，默认使用商品销售额 sum(order_items.price)，但需要在回答中说明不含运费。
当用户明确问“支付金额”“实收金额”“总付款”时，使用 sum(order_payments.payment_value)。
```

### 3.4 去重规则

| 指标类型 | 推荐去重 |
| --- | --- |
| 订单量 | `count(distinct orders.order_id)` |
| 客户数 | `count(distinct customers.customer_unique_id)` |
| 卖家数 | `count(distinct sellers.seller_id)` |
| 商品数 | `count(distinct products.product_id)` |
| 评价订单数 | `count(distinct order_reviews.order_id)` |

## 4. 核心指标口径

### 4.1 交易指标

| 指标名称 | 英文字段建议 | 业务定义 | 计算公式 | 默认过滤 |
| --- | --- | --- | --- | --- |
| 商品销售额 | `sales_amount` | 已送达订单中的商品销售金额，不含运费 | `sum(order_items.price)` | `orders.order_status = 'delivered'` |
| GMV | `gmv_amount` | 默认等同商品销售额；如业务要求含运费，应改用“含运费 GMV” | `sum(order_items.price)` | `orders.order_status = 'delivered'` |
| 含运费 GMV | `gmv_with_freight_amount` | 商品销售额加运费 | `sum(order_items.price + order_items.freight_value)` | `orders.order_status = 'delivered'` |
| 运费金额 | `freight_amount` | 订单商品行运费总额 | `sum(order_items.freight_value)` | `orders.order_status = 'delivered'` |
| 支付金额 | `payment_amount` | 支付表记录的付款金额 | `sum(order_payments.payment_value)` | 默认关联 `orders` 后按业务选择状态 |
| 订单量 | `order_count` | 订单数 | `count(distinct orders.order_id)` | 默认 `orders.order_status = 'delivered'` |
| 订单商品行数 | `order_item_count` | 订单明细行数 | `count(*)` from `order_items` | 默认关联已送达订单 |
| 销售件数 | `sold_item_count` | 订单商品件数；Olist 明细中一行通常表示一个 item | `count(order_items.order_item_id)` | 默认关联已送达订单 |
| 客单价 | `avg_order_value` | 平均每单商品销售额 | `sum(order_items.price) / count(distinct orders.order_id)` | 默认 `orders.order_status = 'delivered'` |
| 平均订单含运费金额 | `avg_order_value_with_freight` | 平均每单商品价格加运费 | `sum(order_items.price + order_items.freight_value) / count(distinct orders.order_id)` | 默认 `orders.order_status = 'delivered'` |
| 平均商品行单价 | `avg_item_price` | 商品行平均价格 | `avg(order_items.price)` | 默认关联已送达订单 |

### 4.2 订单状态指标

| 指标名称 | 英文字段建议 | 业务定义 | 计算公式 | 默认过滤 |
| --- | --- | --- | --- | --- |
| 全部订单量 | `total_order_count` | 所有状态订单数 | `count(distinct orders.order_id)` | 不过滤状态 |
| 已送达订单量 | `delivered_order_count` | 状态为 `delivered` 的订单数 | `count(distinct case when order_status = 'delivered' then order_id end)` | 不过滤状态 |
| 取消订单量 | `canceled_order_count` | 状态为 `canceled` 的订单数 | `count(distinct case when order_status = 'canceled' then order_id end)` | 不过滤状态 |
| 订单送达率 | `delivery_rate` | 已送达订单占全部订单比例 | `delivered_order_count / total_order_count` | 不过滤状态 |
| 订单取消率 | `cancel_rate` | 取消订单占全部订单比例 | `canceled_order_count / total_order_count` | 不过滤状态 |

### 4.3 客户指标

| 指标名称 | 英文字段建议 | 业务定义 | 计算公式 | 默认过滤 |
| --- | --- | --- | --- | --- |
| 下单客户数 | `customer_count` | 下单客户去重数，按匿名唯一客户 ID 去重 | `count(distinct customers.customer_unique_id)` | 默认关联已送达订单 |
| 订单客户记录数 | `customer_order_record_count` | `customer_id` 去重数，接近订单客户记录数 | `count(distinct customers.customer_id)` | 默认关联已送达订单 |
| 人均订单数 | `orders_per_customer` | 平均每位客户订单数 | `count(distinct orders.order_id) / count(distinct customers.customer_unique_id)` | 默认关联已送达订单 |
| 人均商品销售额 | `sales_amount_per_customer` | 平均每位客户贡献的商品销售额 | `sum(order_items.price) / count(distinct customers.customer_unique_id)` | 默认关联已送达订单 |

注意：

```text
客户分析优先使用 customer_unique_id。
customer_id 更接近订单级匿名客户记录，同一个真实客户可能有多个 customer_id。
```

### 4.4 商品和品类指标

| 指标名称 | 英文字段建议 | 业务定义 | 计算公式 | 默认过滤 |
| --- | --- | --- | --- | --- |
| 商品数 | `product_count` | 商品 ID 去重数 | `count(distinct products.product_id)` | 按分析场景 |
| 品类数 | `category_count` | 商品品类去重数 | `count(distinct coalesce(product_category_name_english, product_category_name))` | 按分析场景 |
| 品类商品销售额 | `category_sales_amount` | 品类维度商品销售额 | `sum(order_items.price)` by category | 默认关联已送达订单 |
| 品类订单量 | `category_order_count` | 品类维度订单数 | `count(distinct orders.order_id)` by category | 默认关联已送达订单 |
| 品类客单价 | `category_avg_order_value` | 品类维度平均每单商品销售额 | `sum(order_items.price) / count(distinct orders.order_id)` by category | 默认关联已送达订单 |
| 平均商品重量 | `avg_product_weight_g` | 商品平均重量，单位克 | `avg(products.product_weight_g)` | 排除空值 |
| 平均商品体积 | `avg_product_volume_cm3` | 商品长宽高乘积的平均值 | `avg(product_length_cm * product_height_cm * product_width_cm)` | 排除空值 |

品类展示规则：

```sql
coalesce(
    product_category_translation.product_category_name_english,
    products.product_category_name,
    'unknown'
) AS category
```

### 4.5 卖家指标

| 指标名称 | 英文字段建议 | 业务定义 | 计算公式 | 默认过滤 |
| --- | --- | --- | --- | --- |
| 卖家数 | `seller_count` | 卖家 ID 去重数 | `count(distinct sellers.seller_id)` | 按分析场景 |
| 活跃卖家数 | `active_seller_count` | 有已送达订单商品行的卖家数 | `count(distinct order_items.seller_id)` | 默认关联已送达订单 |
| 卖家商品销售额 | `seller_sales_amount` | 卖家维度商品销售额 | `sum(order_items.price)` by seller | 默认关联已送达订单 |
| 卖家订单量 | `seller_order_count` | 卖家维度订单数 | `count(distinct order_items.order_id)` by seller | 默认关联已送达订单 |
| 卖家平均客单价 | `seller_avg_order_value` | 卖家平均每单商品销售额 | `sum(order_items.price) / count(distinct order_items.order_id)` | 默认关联已送达订单 |
| 卖家平均评分 | `seller_avg_review_score` | 卖家相关订单平均评价分 | `avg(order_reviews.review_score)` | 默认关联已送达订单和评价 |

注意：

```text
一个订单可能包含多个卖家的商品。
卖家维度订单量使用 order_items.order_id 去重，但不同卖家共享同一个 order_id 时，各卖家都会计入该订单。
```

### 4.6 支付指标

| 指标名称 | 英文字段建议 | 业务定义 | 计算公式 | 默认过滤 |
| --- | --- | --- | --- | --- |
| 支付记录数 | `payment_record_count` | 支付表记录行数 | `count(*)` from `order_payments` | 按分析场景 |
| 支付订单数 | `paid_order_count` | 有支付记录的订单数 | `count(distinct order_payments.order_id)` | 按分析场景 |
| 支付金额 | `payment_amount` | 支付记录金额合计 | `sum(order_payments.payment_value)` | 按分析场景 |
| 平均支付金额 | `avg_payment_amount` | 平均每个支付订单的付款金额 | `sum(payment_value) / count(distinct order_id)` | 按分析场景 |
| 分期订单数 | `installment_order_count` | 分期数大于 1 的支付订单数 | `count(distinct case when payment_installments > 1 then order_id end)` | 按分析场景 |
| 分期订单占比 | `installment_order_rate` | 分期订单占支付订单比例 | `installment_order_count / paid_order_count` | 按分析场景 |
| 支付方式占比 | `payment_type_share` | 各支付方式支付订单数占比 | `count(distinct order_id) by payment_type / paid_order_count` | 按分析场景 |

注意：

```text
一个订单可能有多条支付记录。
计算支付方式占比时，如按订单去重，需说明一个订单多支付方式的处理方式。
默认可以按支付记录统计，也可以按订单统计，但二者不能混用。
```

### 4.7 物流和履约指标

| 指标名称 | 英文字段建议 | 业务定义 | 计算公式 | 默认过滤 |
| --- | --- | --- | --- | --- |
| 审批耗时 | `approval_hours` | 从下单到审批的小时数 | `extract(epoch from order_approved_at - order_purchase_timestamp) / 3600` | 已审批订单 |
| 发货耗时 | `ship_hours` | 从审批到交给物流的小时数 | `extract(epoch from order_delivered_carrier_date - order_approved_at) / 3600` | 已发货订单 |
| 配送耗时 | `delivery_hours` | 从交给物流到客户签收的小时数 | `extract(epoch from order_delivered_customer_date - order_delivered_carrier_date) / 3600` | 已送达订单 |
| 履约总时长 | `fulfillment_days` | 从下单到客户签收的天数 | `extract(epoch from order_delivered_customer_date - order_purchase_timestamp) / 86400` | 已送达订单 |
| 平均履约总时长 | `avg_fulfillment_days` | 已送达订单平均履约天数 | `avg(fulfillment_days)` | 已送达订单 |
| 预计送达提前/延迟天数 | `delivery_delay_days` | 实际送达日期减预计送达日期，正数表示延迟 | `extract(epoch from order_delivered_customer_date - order_estimated_delivery_date) / 86400` | 已送达订单 |
| 准时送达订单量 | `on_time_order_count` | 实际送达时间小于等于预计送达时间的订单数 | `count(distinct case when delivered_customer_date <= estimated_delivery_date then order_id end)` | 已送达订单 |
| 延迟送达订单量 | `late_order_count` | 实际送达时间晚于预计送达时间的订单数 | `count(distinct case when delivered_customer_date > estimated_delivery_date then order_id end)` | 已送达订单 |
| 延迟送达率 | `late_delivery_rate` | 延迟送达订单占已送达订单比例 | `late_order_count / delivered_order_count` | 已送达订单 |

注意：

```text
物流耗时计算必须排除相关时间字段为空的订单。
趋势分析默认用下单时间分组；物流过程分析可按送达时间分组，但需在结果中说明。
```

### 4.8 评价指标

| 指标名称 | 英文字段建议 | 业务定义 | 计算公式 | 默认过滤 |
| --- | --- | --- | --- | --- |
| 评价数 | `review_count` | 评价记录数 | `count(*)` from `order_reviews` | 有评价记录 |
| 评价订单数 | `reviewed_order_count` | 有评价的订单数 | `count(distinct order_reviews.order_id)` | 有评价记录 |
| 平均评分 | `avg_review_score` | 客户评价分平均值 | `avg(order_reviews.review_score)` | 有评价记录 |
| 低评分评价数 | `low_score_review_count` | 评分小于等于 2 的评价数 | `count(case when review_score <= 2 then 1 end)` | 有评价记录 |
| 低评分订单数 | `low_score_order_count` | 有低评分评价的订单数 | `count(distinct case when review_score <= 2 then order_id end)` | 有评价记录 |
| 低评分率 | `low_score_rate` | 低评分评价数占评价数比例 | `low_score_review_count / review_count` | 有评价记录 |
| 好评评价数 | `high_score_review_count` | 评分大于等于 4 的评价数 | `count(case when review_score >= 4 then 1 end)` | 有评价记录 |
| 好评率 | `high_score_rate` | 好评评价数占评价数比例 | `high_score_review_count / review_count` | 有评价记录 |
| 有文字评价数 | `review_with_comment_count` | 标题或正文非空的评价数 | `count(case when title/message is not null then 1 end)` | 有评价记录 |
| 评价回复耗时 | `review_answer_hours` | 从评价创建到评价回复的小时数 | `extract(epoch from review_answer_timestamp - review_creation_date) / 3600` | 回复时间非空 |

注意：

```text
评价表可能存在一个订单多条评价记录。
订单级评价分析应按 order_id 去重，评价内容分析可按 review_id 统计。
```

### 4.9 区域指标

| 指标名称 | 英文字段建议 | 业务定义 | 计算公式 | 默认过滤 |
| --- | --- | --- | --- | --- |
| 客户州销售额 | `customer_state_sales_amount` | 按客户州统计商品销售额 | `sum(order_items.price)` by `customers.customer_state` | 默认已送达订单 |
| 客户城市销售额 | `customer_city_sales_amount` | 按客户城市统计商品销售额 | `sum(order_items.price)` by `customers.customer_city` | 默认已送达订单 |
| 卖家州销售额 | `seller_state_sales_amount` | 按卖家州统计商品销售额 | `sum(order_items.price)` by `sellers.seller_state` | 默认已送达订单 |
| 客户州订单量 | `customer_state_order_count` | 按客户州统计订单量 | `count(distinct orders.order_id)` by `customers.customer_state` | 默认已送达订单 |
| 卖家州订单量 | `seller_state_order_count` | 按卖家州统计订单量 | `count(distinct order_items.order_id)` by `sellers.seller_state` | 默认已送达订单 |

注意：

```text
客户地区来自 customers，卖家地区来自 sellers。
不要把客户州和卖家州混为同一个“地区”口径。
```

## 5. 常用维度口径

| 维度名称 | 字段 / 表达式 | 说明 |
| --- | --- | --- |
| 日期 | `date(order_purchase_timestamp)` | 默认下单日期 |
| 月份 | `date_trunc('month', order_purchase_timestamp)` | 趋势分析常用 |
| 年份 | `extract(year from order_purchase_timestamp)` | 年度分析 |
| 订单状态 | `orders.order_status` | 状态分布、取消率分析 |
| 商品品类 | `coalesce(t.product_category_name_english, p.product_category_name, 'unknown')` | 默认优先英文品类 |
| 客户州 | `customers.customer_state` | 客户所在地州 |
| 客户城市 | `customers.customer_city` | 客户所在地城市 |
| 卖家州 | `sellers.seller_state` | 卖家所在地州 |
| 卖家城市 | `sellers.seller_city` | 卖家所在地城市 |
| 支付方式 | `order_payments.payment_type` | 信用卡、票据、借记卡等 |
| 分期数 | `order_payments.payment_installments` | 支付分期期数 |
| 评分 | `order_reviews.review_score` | 1-5 分 |

## 6. Join 规则

### 6.1 推荐 Join 图

```text
orders
├── customers: orders.customer_id = customers.customer_id
├── order_items: orders.order_id = order_items.order_id
│   ├── products: order_items.product_id = products.product_id
│   │   └── product_category_translation:
│   │       products.product_category_name = product_category_translation.product_category_name
│   └── sellers: order_items.seller_id = sellers.seller_id
├── order_payments: orders.order_id = order_payments.order_id
└── order_reviews: orders.order_id = order_reviews.order_id
```

### 6.2 Join 规则表

| 规则名称 | 左表 | 右表 | Join 条件 | 推荐 Join 类型 | 用途 |
| --- | --- | --- | --- | --- | --- |
| 订单-客户 | `olist.orders` | `olist.customers` | `orders.customer_id = customers.customer_id` | `LEFT JOIN` | 客户地区、客户数 |
| 订单-明细 | `olist.orders` | `olist.order_items` | `orders.order_id = order_items.order_id` | `INNER JOIN` 或 `LEFT JOIN` | 销售额、商品、卖家分析 |
| 明细-商品 | `olist.order_items` | `olist.products` | `order_items.product_id = products.product_id` | `LEFT JOIN` | 商品和品类分析 |
| 商品-品类翻译 | `olist.products` | `olist.product_category_translation` | `products.product_category_name = product_category_translation.product_category_name` | `LEFT JOIN` | 英文品类展示 |
| 明细-卖家 | `olist.order_items` | `olist.sellers` | `order_items.seller_id = sellers.seller_id` | `LEFT JOIN` | 卖家分析 |
| 订单-支付 | `olist.orders` | `olist.order_payments` | `orders.order_id = order_payments.order_id` | `LEFT JOIN` | 支付方式、支付金额 |
| 订单-评价 | `olist.orders` | `olist.order_reviews` | `orders.order_id = order_reviews.order_id` | `LEFT JOIN` | 评分、评价、低评分分析 |
| 客户-地理位置 | `olist.customers` | `olist.geolocation` | `customers.customer_zip_code_prefix = geolocation.geolocation_zip_code_prefix` | 谨慎使用 | 经纬度分析，需要处理一对多 |
| 卖家-地理位置 | `olist.sellers` | `olist.geolocation` | `sellers.seller_zip_code_prefix = geolocation.geolocation_zip_code_prefix` | 谨慎使用 | 卖家经纬度分析，需要处理一对多 |

### 6.3 防重复聚合规则

`order_items`、`order_payments`、`order_reviews` 都可能与 `orders` 形成一对多关系。多张一对多事实表同时 Join 时，容易造成金额被重复放大。

风险示例：

```text
orders
-> order_items 一单多商品行
-> order_payments 一单多支付记录
-> order_reviews 一单多评价

如果直接三表 Join 后 sum(order_items.price)，商品销售额可能被支付记录数或评价记录数放大。
```

推荐做法：

- 商品销售额、运费、商品行数：先在 `order_items` 粒度计算。
- 支付金额：先在 `order_payments` 按 `order_id` 聚合后再 Join。
- 评价指标：先在 `order_reviews` 按 `order_id` 或 `review_id` 粒度明确后再 Join。
- 订单量始终使用 `count(distinct orders.order_id)` 或先聚合订单粒度。

## 7. SQL 模板

### 7.1 Top 10 品类商品销售额

```sql
SELECT
    coalesce(t.product_category_name_english, p.product_category_name, 'unknown') AS category,
    round(sum(oi.price), 2) AS sales_amount,
    count(DISTINCT o.order_id) AS order_count,
    round(sum(oi.price) / nullif(count(DISTINCT o.order_id), 0), 2) AS avg_order_value
FROM olist.orders AS o
JOIN olist.order_items AS oi
    ON oi.order_id = o.order_id
LEFT JOIN olist.products AS p
    ON p.product_id = oi.product_id
LEFT JOIN olist.product_category_translation AS t
    ON t.product_category_name = p.product_category_name
WHERE o.order_status = 'delivered'
GROUP BY 1
ORDER BY sales_amount DESC
LIMIT 10;
```

### 7.2 月度销售额趋势

```sql
SELECT
    date_trunc('month', o.order_purchase_timestamp)::date AS month,
    round(sum(oi.price), 2) AS sales_amount,
    count(DISTINCT o.order_id) AS order_count
FROM olist.orders AS o
JOIN olist.order_items AS oi
    ON oi.order_id = o.order_id
WHERE o.order_status = 'delivered'
GROUP BY 1
ORDER BY 1;
```

### 7.3 客户州销售额排行

```sql
SELECT
    c.customer_state,
    round(sum(oi.price), 2) AS sales_amount,
    count(DISTINCT o.order_id) AS order_count,
    count(DISTINCT c.customer_unique_id) AS customer_count
FROM olist.orders AS o
JOIN olist.order_items AS oi
    ON oi.order_id = o.order_id
LEFT JOIN olist.customers AS c
    ON c.customer_id = o.customer_id
WHERE o.order_status = 'delivered'
GROUP BY 1
ORDER BY sales_amount DESC;
```

### 7.4 卖家履约表现

```sql
SELECT
    oi.seller_id,
    s.seller_state,
    count(DISTINCT o.order_id) AS order_count,
    round(avg(extract(epoch FROM o.order_delivered_customer_date - o.order_purchase_timestamp) / 86400), 2) AS avg_fulfillment_days,
    round(avg(CASE WHEN o.order_delivered_customer_date > o.order_estimated_delivery_date THEN 1 ELSE 0 END), 4) AS late_delivery_rate
FROM olist.orders AS o
JOIN olist.order_items AS oi
    ON oi.order_id = o.order_id
LEFT JOIN olist.sellers AS s
    ON s.seller_id = oi.seller_id
WHERE o.order_status = 'delivered'
  AND o.order_delivered_customer_date IS NOT NULL
  AND o.order_estimated_delivery_date IS NOT NULL
GROUP BY 1, 2
HAVING count(DISTINCT o.order_id) >= 20
ORDER BY late_delivery_rate DESC, avg_fulfillment_days DESC
LIMIT 20;
```

### 7.5 低评分订单原因分析基础查询

```sql
SELECT
    r.review_score,
    count(DISTINCT o.order_id) AS order_count,
    round(avg(extract(epoch FROM o.order_delivered_customer_date - o.order_purchase_timestamp) / 86400), 2) AS avg_fulfillment_days,
    round(avg(CASE WHEN o.order_delivered_customer_date > o.order_estimated_delivery_date THEN 1 ELSE 0 END), 4) AS late_delivery_rate,
    round(avg(oi.freight_value), 2) AS avg_freight_value
FROM olist.orders AS o
JOIN olist.order_items AS oi
    ON oi.order_id = o.order_id
LEFT JOIN olist.order_reviews AS r
    ON r.order_id = o.order_id
WHERE o.order_status = 'delivered'
  AND r.review_score IS NOT NULL
GROUP BY 1
ORDER BY 1;
```

### 7.6 支付方式占比

```sql
WITH payment_by_order AS (
    SELECT
        order_id,
        payment_type,
        sum(payment_value) AS payment_amount
    FROM olist.order_payments
    GROUP BY 1, 2
)
SELECT
    payment_type,
    count(DISTINCT order_id) AS paid_order_count,
    round(sum(payment_amount), 2) AS payment_amount,
    round(
        count(DISTINCT order_id)::numeric
        / nullif(sum(count(DISTINCT order_id)) OVER (), 0),
        4
    ) AS paid_order_share
FROM payment_by_order
GROUP BY 1
ORDER BY paid_order_count DESC;
```

## 8. 指标同义词

| 用户说法 | 默认匹配指标 | 说明 |
| --- | --- | --- |
| 销售额 | 商品销售额 `sales_amount` | 默认不含运费 |
| GMV | GMV `gmv_amount` | 默认等同商品销售额 |
| 成交金额 | 商品销售额 `sales_amount` | 如用户强调付款，切到支付金额 |

| 好评率 | 好评率 `high_score_rate` | 默认评分 >= 4 |

## 10. 需要人工确认的口径

这些口径会显著影响分析结果，建议在语义层中标记为高风险资产，需要人工审核后发布。

| 待确认项 | 当前建议 | 风险 |
| --- | --- | --- |
| GMV 是否包含运费 | 第一阶段默认不含运费，即 `sum(order_items.price)` | 不同团队可能把含运费金额也叫 GMV |
| 销售额是否只统计已送达订单 | 第一阶段默认只统计 `delivered` | 如做下单意向分析，可能需要包含其他状态 |
| 支付金额和商品销售额的优先级 | 用户说“销售额”用商品价格，用户说“支付/付款”用支付表 | 二者金额可能不一致 |
| 低评分阈值 | 默认 `review_score <= 2` | 有些业务把 3 分也视为中差评 |
| 好评阈值 | 默认 `review_score >= 4` | 可按业务调整 |
| 卖家履约差的最小订单量 | 示例 SQL 使用 `HAVING count(distinct order_id) >= 20` | 样本过小会导致排名不稳定 |
| 地理位置表 Join | 暂不作为默认 Join | 邮编经纬度存在一对多，直接 Join 可能放大数据 |

## 12. 对 AI 分析回答的要求

当 AI 使用本文档中的指标时，最终回答需要包含：

- 使用的指标名称和口径说明。
- 使用的数据表和关键字段。
- 默认过滤条件，例如是否只统计已送达订单。
- 查询结果的时间范围和维度。
- 如果指标存在多口径可能，需要说明本次采用的口径。
- 如果 Join 可能导致重复聚合，需要说明已先聚合或已去重。
- 对样本量较小的排名，需要提示稳定性风险。

示例说明：

```text
本次“销售额”采用商品销售额口径：sum(olist.order_items.price)，不包含运费；
订单范围默认过滤为 orders.order_status = 'delivered'；
品类名称优先使用 product_category_name_translation.product_category_name_english。
```
