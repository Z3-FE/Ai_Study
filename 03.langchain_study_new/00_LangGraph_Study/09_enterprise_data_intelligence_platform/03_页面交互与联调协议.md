# 后端接口文档

## 1. 文档目标

本文档用于前端直接联调 FastAPI 后端接口。

当前阶段不做 BFF 层，不做 Next.js API 代理，不做字段兼容转换。

前端可以先使用：

```ts
const data: any = await response.json();
```

等接口稳定后，再逐步补充 TypeScript 类型。

## 2. 联调约定

### 2.1 基础地址

本地后端：

```text
http://127.0.0.1:8000
```

接口统一前缀：

```text
/api
```

示例：

```text
GET http://127.0.0.1:8000/api/conversations
```

### 2.2 字段格式

以后端返回字段格式为主。

前端不做字段兼容，不做命名转换，不同时支持两套格式。

也就是说：

```text
后端返回 thread_id
前端就使用 data.thread_id
```

不要改成：

```text
threadId
```

后端当前主要使用 `snake_case`：

```json
{
  "conversation_id": "conv_xxx",
  "thread_id": "thread_xxx",
  "created_at": "2026-06-08T10:00:00Z"
}
```

前端按原样消费。

### 2.3 请求格式

所有 JSON 接口默认使用：

```http
Content-Type: application/json
```

当前阶段暂不做登录鉴权。

### 2.4 响应格式

成功响应直接返回业务数据对象。

示例：

```json
{
  "conversation_id": "conv_123",
  "thread_id": "thread_123",
  "conversation": {}
}
```

失败响应遵循 FastAPI 默认格式。

示例：

```json
{
  "detail": "Conversation not found"
}
```

参数校验失败通常是：

```json
{
  "detail": []
}
```

### 2.5 前端调用建议

第一阶段前端可以先封装一个极简请求函数：

```ts
const API_BASE_URL = "http://127.0.0.1:8000";

async function request(path: string, options?: RequestInit): Promise<any> {
  const response = await fetch(`${API_BASE_URL}${path}`, {
    ...options,
    headers: {
      "Content-Type": "application/json",
      ...(options?.headers ?? {}),
    },
  });

  const data: any = await response.json();

  if (!response.ok) {
    throw new Error(data?.detail ?? "接口请求失败");
  }

  return data;
}
```

注意：这里返回 `any` 是当前阶段的简化策略，不代表最终生产项目都应该长期使用 `any`。

## 7. 推荐联调顺序

### 7.1 新建会话主流程

```text
GET /api/app/bootstrap
-> POST /api/conversations
-> GET /api/conversations/{conversation_id}
-> GET /api/conversations/{conversation_id}/messages
```

### 7.2 上传指标口径并审核

```text
POST /api/corpus/documents
-> POST /api/corpus/documents/{document_id}/extract-semantic-assets
-> PATCH /api/semantic/drafts/{draft_id}
-> POST /api/semantic/drafts/{draft_id}/adopt
-> POST /api/conversations
```

### 7.3 发布到全局语义资产

```text
POST /api/semantic/drafts/{draft_id}/publish?mode=sync
-> GET /api/semantic/assets/metrics
-> GET /api/semantic/assets/dimensions
-> GET /api/semantic/assets/glossary
-> GET /api/vector/jobs
```

### 7.4 语义资产四页联调

```text
GET /api/semantic/assets/metrics
-> 渲染指标管理页

GET /api/semantic/assets/dimensions
-> 渲染维度管理页

GET /api/semantic/assets/datasets
-> 渲染数据集说明页

GET /api/semantic/assets/glossary
-> 渲染业务术语页
```

说明：

这一步相当于用语义资产四页测试后端 API 是否可用。

前端不经过 BFF，不请求 Next.js API route，直接请求 FastAPI。

### 9.4 读取语义资产四页数据

```ts
const API_BASE_URL = "http://127.0.0.1:8000";

async function apiGet(path: string): Promise<any> {
  const response = await fetch(`${API_BASE_URL}${path}`, {
    headers: {
      "Content-Type": "application/json",
    },
  });

  const data: any = await response.json();

  if (!response.ok) {
    throw new Error(data?.detail ?? "接口请求失败");
  }

  return data;
}

const metricsData: any = await apiGet("/api/semantic/assets/metrics");
console.log(metricsData.metrics[0].metric_id);
console.log(metricsData.metrics[0].sql_expression);

const dimensionsData: any = await apiGet("/api/semantic/assets/dimensions");
console.log(dimensionsData.dimensions[0].field_mapping);

const datasetsData: any = await apiGet("/api/semantic/assets/datasets");
console.log(datasetsData.datasets[0].fields);

const glossaryData: any = await apiGet("/api/semantic/assets/glossary");
console.log(glossaryData.glossary[0].mapping_target);
```

注意：

- 前端直接消费后端字段，例如 `metric_id`、`sql_expression`、`field_mapping`、`mapping_target`。
- 不转换为 `metricId`、`sqlExpression`、`fieldMapping`、`mappingTarget`。
